📊 ML Basics
Supervised vs unsupervised, key metrics, overfitting.
Supervised vs unsupervised
Supervised: Labeled data; model learns input → output. Classification (discrete labels) and regression (continuous). Unsupervised: No labels; find structure: clustering (e.g. K-means), dimensionality reduction (e.g. PCA), anomaly detection. Semi-supervised uses a mix of labeled and unlabeled data.
Metrics and overfitting
Classification: Accuracy, precision, recall, F1, ROC-AUC (especially for imbalanced data). Regression: MSE, MAE, RMSE. Overfitting: Model memorizes training data and does not generalize. Combat with more data, regularization (L1/L2), dropout, early stopping, and cross-validation.
Q: What is bias–variance tradeoff?
Bias: error from wrong assumptions (underfitting). Variance: error from sensitivity to training data (overfitting). Increasing model complexity usually increases variance and decreases bias. You want a balance (e.g. cross-validation, regularization).
Q: When do you use precision vs recall?
Precision: when false positives are costly (e.g. spam filter marking good mail as spam). Recall: when false negatives are costly (e.g. missing a disease). Use F1 when you need a single balance; use precision–recall curves when classes are imbalanced.

🤖 LLMs & RAG
Transformers, prompting, retrieval-augmented generation.
Transformers and LLMs
Transformers use self-attention to model relationships between tokens (no recurrence like RNNs); they scale well and can be parallelized. LLMs are typically decoder-only (e.g. GPT) or encoder-decoder (e.g. T5). Pretraining on large text, then fine-tuning or prompting for tasks.
Prompting and RAG
Prompting: Zero-shot (task in prompt), few-shot (examples in prompt), chain-of-thought (reasoning steps). RAG (Retrieval-Augmented Generation): Query → retrieve relevant documents (embedding + vector store, e.g. similarity search) → pass docs as context to the LLM → generate answer. Reduces hallucination and grounds answers in your data; good for knowledge-heavy or private data.
Q: Why use RAG instead of fine-tuning for knowledge?
RAG lets you update knowledge by changing the document store (no retraining). Fine-tuning is better for changing style or task format but is slower to update and can cause forgetting. RAG is often cheaper and easier to maintain for domain or up-to-date knowledge.
Q: What is chain-of-thought prompting?
Asking the model to output its reasoning step by step before the final answer (e.g. "Let's think step by step"). Often improves accuracy on reasoning tasks because the model explicitly decomposes the problem. Can be combined with few-shot examples that show CoT.

🔄 MLOps
Pipelines, deployment, monitoring.
Pipelines and registry
MLOps brings CI/CD practices to ML: data versioning (e.g. DVC, Delta Lake) so you can reproduce runs; training pipelines (e.g. Kubeflow, Airflow, SageMaker Pipelines) for feature prep, training, and evaluation; model registry (e.g. MLflow) to version and promote models. A/B test new models before full rollout.
Deployment and monitoring
Deployment: Batch (scheduled jobs) vs real-time (API serving). Use model servers (TensorFlow Serving, Triton, TorchServe) or managed platforms (SageMaker, Vertex). Consider latency, throughput, and GPU needs. Monitoring: Data drift (input distribution change), model performance (accuracy, business metrics), latency and errors. Set alerts on SLOs and drift.
Q: What is data drift and why monitor it?
Data drift is when the distribution of input data in production shifts from what the model was trained on. Performance can degrade without any code change. Monitor with statistical tests or embedding distance; retrain or collect new data when drift is detected.

Full quick reference for interviews.
ML
Supervised (classification, regression) vs unsupervised (clustering, PCA). Metrics: precision, recall, F1, ROC-AUC; MSE/MAE. Bias–variance, overfitting (regularization, dropout, early stopping).
