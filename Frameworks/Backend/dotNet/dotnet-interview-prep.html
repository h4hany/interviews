📐 Types: Value vs Reference
The most fundamental C# concept. Understanding the stack, heap, and type system prevents entire classes of bugs.
Value Types vs Reference Types
AspectValue TypesReference Types
Stored onStack (usually)Heap
Examplesint, bool, struct, enum, doubleclass, string, array, interface
AssignmentCopies the valueCopies the reference (pointer)
Default value0 / false / empty structnull
Nullableint? (explicit nullable)Always nullable by default
C#
// Value type — copy semantics
int a = 5;
int b = a;    // b is a COPY
b = 10;
Console.WriteLine(a); // 5 — unchanged

// Reference type — reference semantics
var list1 = new List<int> { 1, 2, 3 };
var list2 = list1;    // list2 points to SAME list
list2.Add(4);
Console.WriteLine(list1.Count); // 4 — list1 changed!

// Boxing — wrapping value type in object (heap allocation!)
int x = 42;
object obj = x;   // boxing — allocates on heap
int y = (int)obj; // unboxing — type check + copy
// Avoid in hot paths — causes GC pressure!

// struct vs class — key differences
struct Point { public int X, Y; }  // value type, stack allocated
record Person(string Name, int Age); // immutable reference type (.NET 5+)
record struct Coord(double Lat, double Lon); // value type record (.NET 6+)
Nullable Reference Types (C# 8+)
C#
// Enable in .csproj: <Nullable>enable</Nullable>
// Now compiler warns about potential null dereferences

string name = "Hany";     // non-nullable — cannot be null
string? nullableName = null; // explicitly nullable

// Null-conditional operator
int? length = nullableName?.Length; // null if nullableName is null

// Null-coalescing
string display = nullableName ?? "Anonymous";
nullableName ??= "Default"; // assign if null

// Null-forgiving operator (use sparingly!)
string guaranteed = nullableName!; // "trust me, not null"
🎯 INTERVIEW QUESTIONS
Q: What is boxing and why should you avoid it?
Boxing converts a value type (int, struct) to object by allocating it on the heap and wrapping it. Unboxing is the reverse. Both are relatively expensive operations: boxing allocates heap memory (GC pressure), unboxing requires a type check and memory copy. Avoid in hot loops, sorting comparisons, or collections that could use generic alternatives (use List<int> instead of ArrayList).
Q: When should you use struct instead of class?
Use struct when: (1) it represents a single value (Point, Color, DateTime), (2) size is small (guideline: under 16 bytes), (3) it's logically immutable, (4) it won't be boxed frequently. Structs avoid heap allocation and GC pressure. Avoid structs for: objects with lots of behavior, large data, objects needing inheritance, objects passed frequently by reference.
Q: What's the difference between record and class in C#?
Records provide value-based equality (two records are equal if all properties are equal), immutability by default, built-in ToString(), and non-destructive mutation via `with` expressions. Classes use reference equality by default and are mutable. Use records for DTOs, value objects, and data transfer. Record structs (C# 10) combine record features with value type semantics.

C#
// Abstract class — IS-A relationship, shared state
public abstract class Shape
{
    public string Color { get; set; } = "Black"; // state OK
    public abstract double Area();              // must override
    public virtual string Describe() =>        // can override
        $"A {Color} shape with area {Area():F2}";
}

// Interface — CAN-DO contract
public interface IDrawable
{
    void Draw();
    void DrawWithColor(string color) => Draw(); // default impl C#8+
}

public class Circle : Shape, IDrawable
{
    private double _radius;
    public Circle(double r) => _radius = r;
    public override double Area() => Math.PI * _radius * _radius;
    public void Draw() => Console.WriteLine("Drawing circle");
}

// sealed — prevents inheritance
public sealed class MySingleton { }

// Covariant return types (C# 9)
public class AnimalFactory {
    public virtual Animal Create() => new Animal();
}
public class DogFactory : AnimalFactory {
    public override Dog Create() => new Dog(); // Dog is Animal ✓
}
🎯 INTERVIEW QUESTIONS
Q: When do you use abstract class vs interface?
Use abstract class when: you have shared implementation/state that subclasses should inherit, you want IS-A relationship, you need a constructor, you want to evolve the API without breaking changes. Use interface when: you want a contract multiple types can implement (CAN-DO), you need multiple "inheritance", you want loose coupling. Rule of thumb: interfaces for contracts, abstract classes for base implementations.
Q: What is the difference between virtual, override, and new?
virtual marks a method as overridable in subclasses. override replaces a virtual/abstract method in a subclass — polymorphic dispatch works correctly. new hides (not overrides) a method — when called through a base reference, the base method runs, not the derived one. This is a common source of bugs. Always prefer virtual/override for intentional polymorphism.

⚡ Async/Await & Tasks
The most-asked C# topic in senior interviews. Tasks, deadlocks, ValueTask, CancellationToken.
Task & async/await Fundamentals
C#
// async methods return Task (void equiv), Task<T> (value), ValueTask<T>
public async Task<string> GetDataAsync(string url)
{
    using var client = new HttpClient();
    var response = await client.GetStringAsync(url);
    return response.ToUpper();
}

// PARALLEL — run concurrently (much faster for independent I/O)
var (users, orders) = await (
    GetUsersAsync(),
    GetOrdersAsync()   // both start at same time
);
// Or: await Task.WhenAll(task1, task2, task3)

// CancellationToken — always pass through for cancellable operations
public async Task<List<string>> SearchAsync(
    string query,
    CancellationToken ct = default)
{
    ct.ThrowIfCancellationRequested();
    return await _db.Items
        .Where(i => i.Name.Contains(query))
        .ToListAsync(ct);  // pass CT to EF Core!
}
ConfigureAwait & Deadlocks
⚠️ Classic .NET Deadlock: Calling .Result or .Wait() on an async method in a synchronization context (like ASP.NET classic, WinForms, WPF) deadlocks. The async method tries to resume on the original thread, but that thread is blocked waiting for it.
C#
// ❌ DEADLOCK in sync context (WinForms, old ASP.NET)
var result = GetDataAsync().Result; // blocks thread → deadlock!

// ✅ Use ConfigureAwait(false) in library code
// "Don't capture the sync context — any thread can resume"
public async Task<string> LibraryMethodAsync()
{
    var data = await _httpClient.GetStringAsync(url)
        .ConfigureAwait(false); // library code: always false
    return data;
}

// ✅ ValueTask — for hot paths (avoids allocation when result is sync)
public ValueTask<int> GetCachedValueAsync()
{
    if (_cache.TryGetValue("key", out int cached))
        return new ValueTask<int>(cached); // no allocation!

    return new ValueTask<int>(FetchFromDbAsync());
}
🎯 INTERVIEW QUESTIONS
Q: Explain the async/await deadlock scenario
In a synchronization context (old ASP.NET, WinForms): async/await captures the current SynchronizationContext. When you await, the continuation is scheduled on the captured context. If you block the context thread with .Result or .Wait(), the continuation can never run because the thread is blocked. Solution: (1) Never use .Result/.Wait() in async code, (2) Use ConfigureAwait(false) in library code, (3) In ASP.NET Core there's no SynchronizationContext, so this is less of an issue.
Q: What is ValueTask and when do you use it?
ValueTask<T> is a struct (value type) alternative to Task<T>. It avoids heap allocation when the result is already available synchronously. Use it for hot-path methods that often return synchronously (like cache lookups). Don't use ValueTask if: it will always be async, it needs to be awaited multiple times, or the code isn't performance-critical — it's more complex to use correctly.
Q: What does ConfigureAwait(false) do?
By default, await captures the SynchronizationContext and resumes on it. ConfigureAwait(false) says "don't capture the context — resume on any available thread." Use in library code (NuGet packages) to avoid deadlocks in consumers who might be in a sync context. In ASP.NET Core application code, it's less critical since there's no SynchronizationContext, but it can still have slight performance benefits.

🔍 LINQ
Language Integrated Query — one of C#'s best features. Deferred execution, IQueryable vs IEnumerable, common operators.
LINQ Fundamentals
C# — LINQ
var users = new List<User> { /* ... */ };

// Method syntax (more common in practice)
var result = users
    .Where(u => u.Age >= 18)
    .OrderBy(u => u.Name)
    .Select(u => new { u.Name, u.Email })
    .Take(10)
    .ToList();  // ← EXECUTES HERE (deferred!)

// GroupBy
var byCity = users
    .GroupBy(u => u.City)
    .Select(g => new { City = g.Key, Count = g.Count() });

// Aggregate (like reduce)
var totalAge = users.Aggregate(0, (acc, u) => acc + u.Age);

// SelectMany — flatten nested collections
var allTags = posts.SelectMany(p => p.Tags).Distinct();
IEnumerable vs IQueryable — Critical Difference
AspectIEnumerable<T>IQueryable<T>
ExecutesIn-memory (C# code)At the data source (SQL)
Used withLINQ to ObjectsEF Core, LINQ to SQL

⚠️ Performance Trap: Calling AsEnumerable() too early forces loading ALL data into memory before filtering. Always filter, sort, and paginate while still IQueryable — only call ToList()/AsEnumerable() at the end.
Deferred Execution
C#
// LINQ queries don't execute until enumerated!
var query = users.Where(u => u.Active); // NO execution yet
// ...code...
foreach (var u in query) { } // EXECUTES HERE
query.ToList();               // EXECUTES HERE (forces eval)
query.Count();                // EXECUTES HERE
query.First();                // EXECUTES HERE

// Benefit: compose queries before executing
var q = _db.Products.Where(p => p.Active);
if (hasFilter) q = q.Where(p => p.Category == filter);
if (sortByPrice) q = q.OrderBy(p => p.Price);
var results = await q.ToListAsync(); // ONE optimized SQL query
🎯 INTERVIEW QUESTIONS
Q: What is deferred execution in LINQ?
LINQ queries don't execute when you define them — they execute when you iterate (foreach) or call a materializing method (ToList, Count, First, etc.). This allows query composition: you can add Where/OrderBy clauses conditionally before executing. It also means queries reflect the state of the data at execution time, not definition time. Be careful: re-iterating a deferred query may execute it multiple times.
Q: Why is using IEnumerable instead of IQueryable bad with EF Core?
IQueryable translates LINQ operations to SQL — filtering happens in the database. If you cast to IEnumerable (e.g., calling AsEnumerable() too early or accepting IEnumerable in a method), EF loads ALL matching rows into memory and then filters in C#. For a million-row table, that's catastrophic. Always keep queries as IQueryable until you're ready to materialize.

📡 Delegates & Events
Type-safe function pointers. The foundation of events, callbacks, and functional programming in C#.
Func, Action, Predicate
C#
// Built-in delegate types
// Action<T> — returns void
Action<string> log = msg => Console.WriteLine(msg);
log("Hello");

// Func<T, TResult> — returns a value
Func<int, int, int> add = (a, b) => a + b;
int sum = add(3, 4); // 7

// Predicate<T> — Func<T, bool>
Predicate<int> isEven = n => n % 2 == 0;

// Multicast delegate — multiple methods
Action<string> pipeline = (string s) => Console.WriteLine("Step 1");
pipeline += (string s) => Console.WriteLine("Step 2");
pipeline("go"); // calls both!

// Events — encapsulated multicast delegate
public class Button
{
    public event EventHandler<ClickEventArgs> Clicked;

    protected virtual void OnClicked(ClickEventArgs e)
        => Clicked?.Invoke(this, e); // null-safe invoke
}
🎯 INTERVIEW QUESTIONS
Q: What's the difference between a delegate and an event?
A delegate is a type-safe function pointer that can be assigned, invoked, and combined. An event is an encapsulated delegate — it restricts external access: from outside the class, you can only += (subscribe) or -= (unsubscribe), not directly invoke or assign. Events prevent external code from firing or resetting all subscriptions. They implement the Observer pattern.
Q: What is a closure in C#?
A closure is a lambda or anonymous method that captures variables from its enclosing scope. The captured variable is kept alive as long as the closure exists. Example: int count = 0; Action inc = () => count++; inc(); // count is 1 — the lambda "closes over" count. This is powerful but be careful: capturing loop variables in old C# captured the variable by reference, not value (same as JavaScript's var problem).

💉 Dependency Injection
The #1 most asked .NET interview topic. Lifetimes, registration, and the pitfalls that cause production bugs.
Service Lifetimes
LifetimeOne instance perUse for
SingletonApp lifetime (one ever)Configuration, caches, stateless services
ScopedHTTP requestDbContext, unit-of-work, request state
TransientEvery injectionLightweight, stateless services
C# — Program.cs / Startup
var builder = WebApplication.CreateBuilder(args);

// Registration
builder.Services.AddSingleton<IConfigService, ConfigService>();
builder.Services.AddScoped<IOrderService, OrderService>();
builder.Services.AddTransient<IEmailSender, SmtpEmailSender>();

// Keyed services (.NET 8 new!)
builder.Services.AddKeyedSingleton<ICache, RedisCache>("redis");
builder.Services.AddKeyedSingleton<ICache, MemoryCache>("memory");

// Usage in controller
public class OrderController(IOrderService orders, // injected!
    [FromKeyedServices("redis")] ICache cache) : ControllerBase
{
    public async Task<IActionResult> GetOrders()
        => Ok(await orders.GetAllAsync());
}
⚠️ Captive Dependency Anti-Pattern: Never inject a Scoped or Transient service into a Singleton. The Singleton lives forever — it captures the Scoped service from its first request and reuses it for all requests, causing state sharing bugs. DI container will throw an exception in development by default (ValidateScopes option).
🎯 INTERVIEW QUESTIONS
Q: What's the difference between Singleton, Scoped, and Transient?
Singleton: one instance for the entire app lifetime — shared by all requests and threads. Must be thread-safe. Use for: configuration readers, in-memory caches, stateless services. Scoped: one instance per HTTP request — new instance per request, shared within a request. Use for: DbContext, unit of work. Transient: new instance every time it's requested from DI. Use for: lightweight, stateless, not shared operations.
Q: What is the Service Locator anti-pattern?
Service Locator means calling IServiceProvider.GetService() directly inside your classes instead of receiving dependencies via constructor injection. It's an anti-pattern because: dependencies are hidden (not visible in constructor), makes unit testing hard (need to set up the whole container), creates implicit coupling to the DI framework. Always prefer constructor injection.

🗄️ Entity Framework Core
EF Core patterns, N+1 problem, AsNoTracking, migrations, and performance best practices.
EF Core Fundamentals
C#
public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }

    public DbSet<User> Users => Set<User>();
    public DbSet<Order> Orders => Set<Order>();

    protected override void OnModelCreating(ModelBuilder mb)
    {
        mb.Entity<User>().HasIndex(u => u.Email).IsUnique();
        mb.Entity<User>().HasMany(u => u.Orders)
            .WithOne(o => o.User)
            .HasForeignKey(o => o.UserId);
    }
}

// Querying best practices
// ✅ AsNoTracking for read-only queries (faster!)
var users = await _db.Users
    .AsNoTracking()        // skip change tracking
    .Where(u => u.Active)
    .Include(u => u.Orders) // eager load (prevent N+1)
    .Select(u => new { u.Name, OrderCount = u.Orders.Count })
    .ToListAsync(ct);

// ✅ Split queries for large includes (.NET 5+)
var users = await _db.Users
    .Include(u => u.Orders).ThenInclude(o => o.Items)
    .AsSplitQuery()  // multiple queries vs cartesian explosion
    .ToListAsync();
🎯 INTERVIEW QUESTIONS
Q: What is AsNoTracking and when should you use it?
AsNoTracking tells EF Core not to track the returned entities in the ChangeTracker. Tracking is needed for SaveChanges() to detect modifications. For read-only queries (GET endpoints, reports), tracking wastes memory and CPU. AsNoTracking improves performance significantly for read-heavy operations — typically 2x faster for large result sets. Use it whenever you don't need to update the returned entities.
Q: How does EF Core handle N+1 and how do you fix it?
EF Core N+1 happens when you access a navigation property inside a loop without loading it upfront. Fix: (1) Include() for eager loading, (2) Select() to project only what's needed, (3) AsSplitQuery() to avoid cartesian explosions with multiple includes, (4) Use Load() explicitly. EF Core 3+ throws exceptions by default for implicit lazy loading (you must explicitly enable it with UseLazyLoadingProxies).

🎯 INTERVIEW QUESTIONS
Q: What are primary constructors in C# 12?
Primary constructors let you declare constructor parameters directly on the class/struct declaration. The parameters are automatically in scope throughout the class body — you don't need to declare private fields manually. They reduce boilerplate for DI injection patterns. Note: unlike record primary constructors, class primary constructors don't automatically create public properties — parameters are just in scope.
Q: What is Native AOT and when would you use it?
Native AOT (Ahead-of-Time compilation) compiles .NET code to native machine code at publish time, not at runtime. Benefits: dramatically faster startup (milliseconds), lower memory usage, smaller deployment size. Limitations: no runtime code generation (reflection limited), no dynamic loading, longer publish times. Ideal for: AWS Lambda/serverless functions, CLI tools, microservices where cold start matters. Not suitable for: apps relying heavily on reflection (EF Core, many libraries).

🔧 Generics & Collections
Type-safe reusable code. List vs Array, Dictionary internals, generic constraints.
Generic Constraints
C#
// Constraints tell the compiler what T can do
public class Repository<T> where T : class, IEntity, new()
// T must be: a class, implement IEntity, have parameterless constructor

// Common constraints:
// where T : class           → reference type
// where T : struct          → value type (non-nullable)
// where T : IComparable<T> → must be comparable
// where T : new()           → must have default constructor
// where T : notnull         → non-nullable (C# 8+)

// Collections comparison
// List<T> — dynamic array (O(1) amortized add, O(n) insert/remove)
// LinkedList<T> — O(1) insert/remove, O(n) access by index
// Dictionary<K,V> — hash table, O(1) average lookup
// SortedDictionary — red-black tree, O(log n)
// HashSet<T> — unique values, O(1) contains check
// Stack<T> — LIFO, ConcurrentStack for thread safety
// Queue<T> — FIFO, ConcurrentQueue for thread safety
🎯 INTERVIEW QUESTIONS
Q: What's the difference between List<T> and Array?
Arrays are fixed-size, allocated contiguously in memory, slightly faster for indexed access. List<T> is a dynamic array — it doubles capacity when full (amortized O(1) adds). Arrays are better when size is known and fixed; List when size varies. For performance-critical code, consider pre-allocating: new List<T>(capacity) to avoid reallocations.
Q: How does Dictionary<K,V> work internally?
Dictionary uses a hash table. Keys are hashed (GetHashCode()), the hash is mapped to a bucket. Collisions are handled by chaining (linked list within bucket). Average O(1) for get/set/contains. Worst case O(n) with many hash collisions. Key requirements: GetHashCode() and Equals() must be correctly implemented for custom keys. Equal objects must have equal hash codes.

🧠 Memory & GC
GC generations, IDisposable pattern, Span<T> for zero-allocation code.
IDisposable & using
C#
// IDisposable pattern — deterministic cleanup
public class FileProcessor : IDisposable
{
    private FileStream _stream;
    private bool _disposed;

    public void Dispose()
    {
        Dispose(true);
        GC.SuppressFinalize(this);
    }

    protected virtual void Dispose(bool disposing)
    {
        if (_disposed) return;
        if (disposing) _stream?.Dispose();
        _disposed = true;
    }
}

// using statement — auto-calls Dispose
using var processor = new FileProcessor(); // C# 8+ scoped using
// Dispose called at end of scope

// Span<T> — zero-allocation slicing (.NET Core+)
string csv = "Hany,30,Cairo";
ReadOnlySpan<char> span = csv.AsSpan();
var name = span[..4];   // "Hany" — NO string allocation!
var age = span[5..7];    // "30" — just a view
🎯 INTERVIEW QUESTIONS
Q: How does .NET's garbage collector work?
.NET GC is generational: Gen 0 (newly allocated, collected most often), Gen 1 (survived Gen 0), Gen 2 (long-lived objects, collected rarely). Most objects die young (Gen 0) — this makes GC very efficient. The LOH (Large Object Heap, objects >85KB) is collected with Gen 2. GC uses mark-and-sweep: mark all reachable objects, sweep unreachable, compact (Gen 0/1). Server GC mode is optimized for throughput; Workstation for responsiveness.
Q: What is Span<T> and why is it important?
Span<T> is a ref struct that represents a contiguous region of memory — a "view" into an array, string, or unmanaged memory without copying. It's stack-only (can't be a class field, can't be boxed, no heap allocation). Use for high-performance parsing, string manipulation, binary protocol parsing. Substring() allocates a new string; AsSpan() slicing doesn't. Critical for high-throughput APIs, parsers, and game/real-time code.

🎯 INTERVIEW QUESTIONS
Q: What are Action Filters and when do you use them?
Action filters run code before/after controller action execution. They implement IActionFilter or IAsyncActionFilter. Use cases: logging, performance metrics, validation, caching, exception handling. Available filters: Authorization (before everything), Resource (before model binding), Action (before/after action), Exception (on unhandled exception), Result (before/after result execution). Applied via [ServiceFilter], [TypeFilter], or globally in AddControllers().
Q: Minimal API vs Controllers — when to use each?
Minimal APIs: simpler projects, microservices, fewer conventions needed, slightly faster startup. Better for: simple CRUD APIs, Lambda/serverless, learning .NET. Controllers: larger apps needing conventions, action filters, complex routing, OpenAPI generation, testing patterns teams already know. In .NET 8, Minimal APIs support most Controller features (filters, authorization, validation) so the gap has closed significantly.

🔗 Middleware & Pipeline
How requests flow through ASP.NET Core. Custom middleware, exception handling, short-circuiting.
Custom Middleware
C#
// Custom middleware class
public class RequestTimingMiddleware(RequestDelegate next)
{
    public async Task InvokeAsync(HttpContext ctx)
    {
        var sw = Stopwatch.StartNew();

        await next(ctx);  // call next middleware in pipeline

        sw.Stop();
        ctx.Response.Headers["X-Response-Time"] =
            $"{sw.ElapsedMilliseconds}ms";
    }
}

// Register in pipeline (order matters!)
app.UseExceptionHandler();         // 1. Must be first
app.UseHttpsRedirection();          // 2.
app.UseMiddleware<RequestTimingMiddleware>(); // 3. Custom
app.UseAuthentication();            // 4. Who are you?
app.UseAuthorization();             // 5. Can you do this?
app.MapControllers();               // 6. Route to action
🎯 INTERVIEW QUESTIONS
Q: What is the ASP.NET Core middleware pipeline?
The pipeline is a chain of middleware components. Each request passes through them in order. Each middleware can: process the request, call next() to pass to the next middleware, process the response on the way back out, or short-circuit (not call next) to return immediately. The order matters: exception handling must be first to catch errors from all subsequent middleware. Authentication before Authorization.
