# Docker Interview Questions

## 1. What is Docker?

**Answer:**
Docker is a containerization platform that packages applications and their dependencies into lightweight, portable
containers. Containers run consistently across different environments (development, staging, production).

## 2. What is the difference between Docker and Virtual Machines?

**Answer:**

- **Docker Containers**: Share the host OS kernel, lightweight, fast startup, less resource overhead.
- **Virtual Machines**: Each VM has its own OS, heavier, slower startup, more resource overhead.

| Feature        | Docker         | Virtual Machines |
|----------------|----------------|------------------|
| OS             | Shares host OS | Full OS per VM   |
| Startup Time   | Seconds        | Minutes          |
| Resource Usage | Low            | High             |
| Isolation      | Process level  | Hardware level   |

## 3. What is a Docker image?

**Answer:**
A Docker image is a read-only template used to create containers. It contains the application code, runtime, libraries,
and dependencies needed to run the application.

## 4. What is a Docker container?

**Answer:**
A Docker container is a running instance of a Docker image. It's an isolated, lightweight process that runs on the host
system.

## 5. What is a Dockerfile?

**Answer:**
A Dockerfile is a text file containing instructions to build a Docker image. It specifies the base image, commands to
run, files to copy, and environment setup.

### Example:

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
```

## 6. What is the difference between `CMD` and `ENTRYPOINT` in Dockerfile?

**Answer:**

- **`CMD`**: Provides default command/arguments for a container. Can be overridden at runtime.
- **`ENTRYPOINT`**: Sets the main command that always runs. Arguments passed at runtime are appended.

### Example:

```dockerfile
# CMD - can be overridden
CMD ["echo", "Hello"]
docker run image echo "Hi"  # Outputs: Hi

# ENTRYPOINT - always runs
ENTRYPOINT ["echo"]
docker run image "Hi"  # Outputs: Hi
```

## 7. What is Docker Compose?

**Answer:**
Docker Compose is a tool for defining and running multi-container Docker applications. It uses a YAML file to configure
services, networks, and volumes.

### Example:

```yaml
version: '3.8'
services:
  web:
    build: .
    ports:
      - "3000:3000"
  db:
    image: postgres:14
    environment:
      POSTGRES_PASSWORD: password
```

## 8. What is the difference between `docker build` and `docker run`?

**Answer:**

- **`docker build`**: Creates a Docker image from a Dockerfile.
- **`docker run`**: Creates and starts a container from an image.

## 9. What is a Docker volume?

**Answer:**
A Docker volume is a mechanism for persisting data generated by containers. Volumes are stored outside the container's
filesystem and survive container deletion.

### Example:

```bash
docker volume create myvolume
docker run -v myvolume:/data myimage
```

## 10. What is the difference between bind mounts and volumes?

**Answer:**

- **Bind Mounts**: Mount a specific file/directory from the host into the container. Path is tied to host.
- **Volumes**: Managed by Docker, stored in Docker's directory, portable across hosts.

## 11. What is Docker networking?

**Answer:**
Docker networking allows containers to communicate with each other and with the outside world. Docker provides bridge,
host, overlay, and macvlan network drivers.

## 12. What is the difference between `EXPOSE` and `-p` flag?

**Answer:**

- **`EXPOSE`**: Documents which ports the container listens on (informational, doesn't publish).
- **`-p`**: Actually publishes and maps container ports to host ports.

### Example:

```dockerfile
EXPOSE 3000  # Documentation only
```

```bash
docker run -p 3000:3000 myimage  # Actually maps port
```

## 13. What is a multi-stage build in Docker?

**Answer:**
Multi-stage builds allow you to use multiple FROM statements in a Dockerfile to create smaller final images by copying
artifacts from one stage to another.

### Example:

```dockerfile
# Build stage
FROM node:18 AS builder
WORKDIR /app
COPY . .
RUN npm run build

# Production stage
FROM node:18-alpine
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY package*.json ./
RUN npm install --production
CMD ["node", "dist/server.js"]
```

## 14. What is Docker Hub?

**Answer:**
Docker Hub is a cloud-based registry service for storing and sharing Docker images. It's the default registry for
Docker.

## 15. What is the difference between `docker stop` and `docker kill`?

**Answer:**

- **`docker stop`**: Gracefully stops a container (sends SIGTERM, then SIGKILL after timeout).
- **`docker kill`**: Immediately kills a container (sends SIGKILL).

## 16. What is `docker exec`?

**Answer:**
`docker exec` runs a command in a running container without stopping it.

### Example:

```bash
docker exec -it container_name /bin/bash
```

## 17. What is the difference between `docker ps` and `docker ps -a`?

**Answer:**

- **`docker ps`**: Shows only running containers.
- **`docker ps -a`**: Shows all containers (running and stopped).

## 18. What is Docker Swarm?

**Answer:**
Docker Swarm is Docker's native clustering and orchestration tool for managing a cluster of Docker hosts.

## 19. What is the `.dockerignore` file?

**Answer:**
`.dockerignore` is similar to `.gitignore` - it specifies files and directories to exclude from the Docker build
context, reducing build time and image size.

## 20. What is the difference between `COPY` and `ADD` in Dockerfile?

**Answer:**

- **`COPY`**: Copies files/directories from host to container (simpler, recommended).
- **`ADD`**: Can copy and extract archives, download from URLs (more features but less predictable).

## 21. What is Docker layer caching?

**Answer:**
Docker caches each layer in a Dockerfile. If a layer hasn't changed, Docker reuses the cached layer, speeding up builds.

## 22. How do you optimize Docker image size?

**Answer:**

- Use multi-stage builds
- Use minimal base images (Alpine Linux)
- Combine RUN commands to reduce layers
- Remove unnecessary files
- Use `.dockerignore`
- Don't install unnecessary packages

## 23. What is the difference between `docker commit` and `docker build`?

**Answer:**

- **`docker commit`**: Creates an image from a running container's changes (not recommended for production).
- **`docker build`**: Creates an image from a Dockerfile (reproducible, recommended).

## 24. What is Docker healthcheck?

**Answer:**
A healthcheck is a command that Docker runs to check if a container is still working. It helps Docker know if a
container is healthy.

### Example:

```dockerfile
HEALTHCHECK --interval=30s --timeout=3s \
  CMD curl -f http://localhost:3000/health || exit 1
```

## 25. What is the difference between `docker rm` and `docker rmi`?

**Answer:**

- **`docker rm`**: Removes one or more containers.
- **`docker rmi`**: Removes one or more images.

## 26. What is Docker overlay network?

**Answer:**
An overlay network enables communication between containers running on different Docker hosts, useful for Docker Swarm
or multi-host setups.

## 27. What is the difference between `docker run` and `docker start`?

**Answer:**

- **`docker run`**: Creates and starts a new container.
- **`docker start`**: Starts an existing stopped container.

## 28. What is Docker context?

**Answer:**
Docker context defines where Docker commands are executed. It can point to local Docker daemon or remote Docker hosts.

## 29. How do you handle secrets in Docker?

**Answer:**

- Use Docker secrets (Docker Swarm)
- Use environment variables (not secure for sensitive data)
- Use external secret management (Vault, AWS Secrets Manager)
- Use Docker build secrets (BuildKit)

## 30. What is the difference between `docker-compose up` and `docker-compose up -d`?

**Answer:**

- **`docker-compose up`**: Starts containers in foreground (logs visible).
- **`docker-compose up -d`**: Starts containers in detached mode (background).

## 31. What is Docker BuildKit?

**Answer:**
BuildKit is an improved backend for building Docker images with better performance, caching, and parallel builds.

## 32. What is the difference between `docker pull` and `docker push`?

**Answer:**

- **`docker pull`**: Downloads an image from a registry.
- **`docker push`**: Uploads an image to a registry.

## 33. How do you debug a Docker container?

**Answer:**

- Use `docker logs` to view container logs
- Use `docker exec` to enter a running container
- Use `docker inspect` to view container details
- Use `docker stats` to monitor resource usage

## 34. What is the difference between `docker-compose build` and `docker-compose up --build`?

**Answer:**

- **`docker-compose build`**: Only builds images, doesn't start containers.
- **`docker-compose up --build`**: Builds images and starts containers.

## 35. What is Docker security best practices?

**Answer:**

- Run containers as non-root user
- Use minimal base images
- Scan images for vulnerabilities
- Don't store secrets in images
- Use read-only filesystems where possible
- Limit container capabilities
- Use network policies
- Keep Docker and images updated
