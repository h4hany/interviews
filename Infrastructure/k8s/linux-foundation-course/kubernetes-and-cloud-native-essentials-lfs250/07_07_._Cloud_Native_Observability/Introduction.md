# Chapter Overview

The term _cloud native observability_ may sound like just another buzzword, but many new tools have indeed emerged to address the challenges of monitoring containerized environments. However, the underlying need is real: traditional monitoring approaches are not sufficient for modern, distributed applications.

Conventional server monitoring focuses on basic system metrics such as CPU and memory usage, along with process and operating system logs. In a microservices architecture, however, a key challenge is understanding how requests flow through many interconnected services. This is where tracing comes in: a discipline for tracking requests across distributed systems, especially when multiple services contribute to a single response.

In this chapter, you will explore how container-based infrastructure still relies on collecting metrics and logs, but with significantly different requirements. Modern systems place greater emphasis on network-related issues such as latency, throughput, retries, and application startup time. Additionally, the enormous volume of metrics, logs, and traces generated by distributed systems demands new strategies and tools to manage and interpret this data effectively.
