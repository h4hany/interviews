# Chapter Overview

The process of deploying applications has evolved dramatically over the years. Early software often ran directly on the machine where it was written, later distributed on physical media like floppy disks, CDs, or USB drives. Today, applications are typically committed to a source repository, automatically built into container images, and deployed directly to platforms like Kubernetes.

This shift in how software is delivered is closely tied to the rise of the DevOps movement in the late 2000s—a cultural and technical transformation that reshaped how development and operations teams work together. DevOps introduced new practices and tools that emphasized collaboration, automation, and rapid iteration.

One of the most impactful outcomes was the automation of deployment pipelines, enabling faster, more frequent, and more reliable releases. In this chapter, we’ll explore these practices, commonly known as Continuous Integration and Continuous Delivery (CI/CD), and how they have evolved into modern approaches such as GitOps.
